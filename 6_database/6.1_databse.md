## Database

A pasta **`database`** concentra todos os componentes responsáveis pela persistência dos dados da aplicação. Nela estão os arquivos relacionados à conexão com o banco de dados PostgreSQL, à criação da estrutura das tabelas e às operações de armazenamento e recuperação das informações utilizadas pelo sistema.

Essa organização permite separar a lógica de acesso ao banco das demais camadas da aplicação, tornando o projeto mais modular, organizado e de fácil manutenção.

### Arquivos do projeto

* **conexao.py:** estabelece a conexão entre a aplicação e o banco de dados PostgreSQL utilizando a biblioteca `psycopg2`.
* **banco_de_dados.py:** implementa a classe `BancoDeDados`, responsável pelas operações de persistência dos dados, incluindo cadastro, consulta, atualização, exclusão e registro de empréstimos.
* **schema.sql:** contém os comandos SQL utilizados para criar automaticamente as tabelas do banco de dados necessárias para o funcionamento do sistema.

### Responsabilidades

A camada de banco de dados é responsável por:

* Estabelecer a conexão com o PostgreSQL;
* Criar automaticamente as tabelas do sistema, quando necessário;
* Armazenar os dados de livros, revistas e empréstimos;
* Executar as operações de CRUD (Create, Read, Update e Delete);
* Garantir a persistência e a integridade das informações armazenadas.

A separação da camada de banco de dados facilita a manutenção do sistema, melhora a organização do código e permite que alterações na estrutura do banco sejam realizadas sem impactar diretamente as demais partes da aplicação.
