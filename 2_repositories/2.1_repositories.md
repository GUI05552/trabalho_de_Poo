## Repositories

A pasta **`repositories`** é responsável pela camada de acesso aos dados da aplicação. Nela estão os arquivos que realizam a comunicação entre o sistema e o banco de dados PostgreSQL, executando as operações de persistência e recuperação das informações.

Essa separação permite que as regras de negócio permaneçam independentes da implementação do banco de dados, tornando o projeto mais organizado, modular e de fácil manutenção.

### Arquivos do projeto

* **livro_repository.py:** responsável pelas operações de cadastro, consulta, atualização e exclusão dos livros no banco de dados.
* **usuario_repository.py:** responsável pelo gerenciamento dos dados dos usuários cadastrados.
* **emprestimo_repository.py:** responsável pelo registro e consulta dos empréstimos realizados.

### Responsabilidades

Os repositórios implementam as principais operações de acesso ao banco de dados, tais como:

* Inserção de registros (`INSERT`);
* Consulta de registros (`SELECT`);
* Atualização de registros (`UPDATE`);
* Exclusão de registros (`DELETE`).

Dessa forma, a camada de repositórios atua como intermediária entre as classes do domínio (`models`) e o banco de dados, centralizando todas as operações de persistência e contribuindo para uma arquitetura mais organizada e reutilizável.
